<PageContent componentName="empty_state" type="api">

<LeadParagraph>
  The EmptyState component communicates a lack of content and guides users
  towards action. It supports an illustration, title, description, composed action
  buttons, and additional custom content.
</LeadParagraph>

## Basic Usage

```tsx
import React from 'react';

import {
  EmptyState,
  Button,
  ButtonColor,
  ButtonVariant,
  ButtonGroup,
} from 'react-magma-dom';
import { SearchIcon } from 'react-magma-icons';

export function Example() {
  return (
    <EmptyState
      illustration={<SearchIcon />}
      title="No results found"
      description="Try adjusting your search or filter criteria to find the courses you are looking for."
      actions={
        <ButtonGroup>
          <Button
            color={ButtonColor.primary}
            onClick={() => console.log('Search clicked')}
          >
            Search Again
          </Button>
          <Button
            color={ButtonColor.secondary}
            onClick={() => console.log('Clear clicked')}
          >
            Clear Filters
          </Button>
          <Button
            color={ButtonColor.primary}
            variant={ButtonVariant.link}
            onClick={() => console.log('Browse clicked')}
          >
            Browse All Courses
          </Button>
        </ButtonGroup>
      }
    />
  );
}
```

## Tertiary Action

A link-style button below the primary and secondary buttons. Use it for a lower-priority alternative action.

```tsx
import React from 'react';

import { EmptyState, Button, ButtonColor, ButtonVariant } from 'react-magma-dom';
import { SearchIcon } from 'react-magma-icons';

export function Example() {
  return (
    <EmptyState
      illustration={<SearchIcon />}
      title="No courses found"
      description="We couldn't find any courses matching your search. Try browsing the full catalog."
      actions={
        <Button
          color={ButtonColor.primary}
          variant={ButtonVariant.link}
          onClick={() => console.log('Browse clicked')}
        >
          Browse Course Catalog
        </Button>
      }
    />
  );
}
```

## Danger Mode

Use the `isDanger` prop to display an error state with danger colors on the illustration. The buttons remain the default primary color for consistency.

```tsx
import React from 'react';

import { EmptyState, Button, ButtonColor, ButtonGroup } from 'react-magma-dom';
import { ErrorIcon } from 'react-magma-icons';

export function Example() {
  return (
    <EmptyState
      illustration={<ErrorIcon />}
      isDanger
      title="Unable to load assignments"
      description="We encountered an error while loading your course content. Please try again or contact support."
      actions={
        <ButtonGroup>
          <Button
            color={ButtonColor.primary}
            onClick={() => console.log('Retry clicked')}
          >
            Try Again
          </Button>
          <Button
            color={ButtonColor.secondary}
            onClick={() => console.log('Contact clicked')}
          >
            Contact Support
          </Button>
        </ButtonGroup>
      }
    />
  );
}
```

## Without Illustration

The illustration is optional. You can render an EmptyState with just text and actions.

```tsx
import React from 'react';

import { EmptyState, Button, ButtonColor } from 'react-magma-dom';

export function Example() {
  return (
    <EmptyState
      title="No submissions yet"
      description="Students have not submitted any assignments for this course yet. Check back after the due date."
      actions={
        <Button
          color={ButtonColor.primary}
          onClick={() => console.log('View clicked')}
        >
          View Assignment Details
        </Button>
      }
    />
  );
}
```

## Without Actions

Action buttons are also optional. You can display an informational empty state without any actions.

```tsx
import React from 'react';

import { EmptyState } from 'react-magma-dom';
import { NotificationsIcon } from 'react-magma-icons';

export function Example() {
  return (
    <EmptyState
      illustration={<NotificationsIcon />}
      title="No upcoming deadlines"
      description="You're all caught up! There are no assignments or quizzes due this week."
    />
  );
}
```

## Minimal

At minimum, you can render an EmptyState with just a title.

```tsx
import React from 'react';

import { EmptyState } from 'react-magma-dom';

export function Example() {
  return <EmptyState title="No courses enrolled" />;
}
```

## Loading State

Use the `isLoading` prop to show a loading spinner instead of the content. This is useful when data is being fetched.

```tsx
import React from 'react';

import { EmptyState } from 'react-magma-dom';

export function Example() {
  return <EmptyState isLoading />;
}
```

The loading spinner color adapts to `isDanger` and `isInverse` props.

## Additional Content

Use the `additionalContent` prop to render custom content between the text and action buttons. This is useful for inline search fields, filter chips, or other interactive elements.

```tsx
import React from 'react';

import { EmptyState, Search } from 'react-magma-dom';
import { SearchIcon } from 'react-magma-icons';

export function Example() {
  return (
    <EmptyState
      illustration={<SearchIcon />}
      title="No results found"
      description="Try a different search term or browse by category."
      additionalContent={
        <Search
          isClearable
          placeholder="Search courses..."
          onSearch={value => console.log('Search:', value)}
        />
      }
    />
  );
}
```

You can also use `additionalContent` for browsable categories. Here, clickable `Tag` components let users browse courses by category.

```tsx
import React from 'react';

import { EmptyState, Button, ButtonColor, ButtonVariant, Tag } from 'react-magma-dom';
import { SearchIcon } from 'react-magma-icons';

export function Example() {
  return (
    <EmptyState
      illustration={<SearchIcon />}
      title="No courses found"
      description="Try browsing by category instead."
      actions={
        <Button
          color={ButtonColor.primary}
          variant={ButtonVariant.link}
          onClick={() => console.log('Browse clicked')}
        >
          Browse All Courses
        </Button>
      }
      additionalContent={
        <div style={{ display: 'flex', gap: '8px', flexWrap: 'wrap', justifyContent: 'center' }}>
          <Tag onClick={() => console.log('Biology')}>Biology</Tag>
          <Tag onClick={() => console.log('Chemistry')}>Chemistry</Tag>
          <Tag onClick={() => console.log('Physics')}>Physics</Tag>
          <Tag onClick={() => console.log('Mathematics')}>Mathematics</Tag>
          <Tag onClick={() => console.log('Computer Science')}>Computer Science</Tag>
        </div>
      }
    />
  );
}
```

Additional content can also include form inputs. Here, an `Input` field lets students enter a course access code directly from the empty state.

```tsx
import React from 'react';

import { EmptyState, Button, ButtonColor, Input } from 'react-magma-dom';
import { SchoolIcon } from 'react-magma-icons';

export function Example() {
  return (
    <EmptyState
      illustration={<SchoolIcon />}
      title="No courses yet"
      description="Enter a course access code from your instructor to get started."
      actions={
        <Button
          color={ButtonColor.primary}
          onClick={() => console.log('Join clicked')}
        >
          Join Course
        </Button>
      }
      additionalContent={<Input placeholder="Enter access code" />}
    />
  );
}
```

A `Toggle` can let users opt in to notifications from within an empty state.

```tsx
import React from 'react';

import { EmptyState, Toggle } from 'react-magma-dom';
import { NotificationsIcon } from 'react-magma-icons';

export function Example() {
  const [enabled, setEnabled] = React.useState(false);

  return (
    <EmptyState
      illustration={<NotificationsIcon />}
      title="No new announcements"
      description="There are no announcements for your courses right now."
      additionalContent={
        <Toggle
          checked={enabled}
          labelText="Notify me when new announcements are posted"
          onChange={() => setEnabled(prev => !prev)}
        />
      }
    />
  );
}
```
