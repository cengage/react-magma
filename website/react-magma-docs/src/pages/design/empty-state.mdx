## Usage

Use empty states when there is no data or content to display in a section or page. Empty states help users understand why the area is empty, what they can do to populate it, and that the system is working correctly (it's not an error).

<figure>
  <img src="../../images/empty-state/empty-state-intro.png" alt="Full-page empty state showing a session expired message with a calendar icon, explanatory text prompting the user to sign in again, and actions to sign in or learn more." />
</figure>

Common scenarios include:
- **First-time use** - No data yet (e.g., empty inbox, no saved items)
- **No search results** - Query returned nothing
- **Empty containers** - Empty folders, lists, or tables
- **Cleared content** - All notifications dismissed
- **Error states** - Something went wrong

---

## Principles

### Helpful

Empty states should clearly explain the situation and, when possible, provide guidance on how to resolve it. Users should never feel lost or confused about why content is missing.

### Friendly

The tone should be supportive and encouraging, not blaming the user. Use positive language that guides users toward action.

### Contextual

The message and illustration should relate to the specific context. A "no search results" empty state should look different from a "get started" empty state.

## Anatomy

<figure>
  <img src="../../images/empty-state/empty-state-anatomy.png" alt="Annotated diagram of an empty state with numbered callouts labeling the illustration, title, body text, primary button, and secondary link." />
</figure>

1. **Illustration** - A circular container with an icon that visually represents the empty state
2. **Title** - A brief heading that explains the situation
3. **Body** - Additional context or instructions (optional)
4. **Primary Action** - A button for the main action the user can take (optional)
5. **Secondary Action** - A link button for an alternative action (optional)

## Layouts

### Vertical Layout (Default)

The vertical layout centers content and stacks elements. Use this for most empty states, especially in larger content areas like main pages or modals.

<figure>
  <img src="../../images/empty-state/layout-vertical.png" alt="Card-based empty state displaying a session expired message with an icon, supporting text, and sign-in and learn-more actions stacked vertically." />
</figure>

### Horizontal Layout

The horizontal layout places the illustration beside the content. Use this for narrower containers, sidebars, or when you want a more compact presentation.

<figure>
  <img src="../../images/empty-state/layout-horizontal.png" alt="Card-based empty state with a session expired message, icon aligned to the left, and sign-in and learn-more actions arranged horizontally." />
</figure>

## Color Modes

### Default

Uses the primary brand color (purple/blue) for the illustration. Appropriate for most informational empty states.

<figure>
  <img src="../../images/empty-state/color-default.png" alt="Card displaying a positive session message indicating the user is ready to continue, with a checkmark icon and a single sign-in action." />
</figure>

### Danger

Uses the danger color (red) for error states. Use when the empty state represents a problem that needs attention, such as a failed load or error condition.

<figure>
  <img src="../../images/empty-state/color-danger.png" alt="Card showing a session expired error state using red styling, indicating a higher-severity interruption that requires the user to sign in again." />
</figure>

### Inverse

For use on dark backgrounds. The colors are adjusted for optimal contrast and visibility.

<figure>
  <img src="../../images/empty-state/color-inverse.png" alt="Dark-background empty state showing a no results found message with a search icon, explanatory text, and actions to clear filters or learn more." />
</figure>

## Accessibility

Empty states are designed with accessibility in mind:

- Text maintains a minimum 4.5:1 contrast ratio with the background
- Illustrations are marked as decorative (`aria-hidden`) since they don't convey essential information
- Action buttons are fully keyboard accessible
- All text content is available to screen readers
